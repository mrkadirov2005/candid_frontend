Sprint goal

Set up a production-ready frontend foundation with routing, API layer, React Query configuration, and a clean project structure so feature development can start fast and safely.

Scope (what will be delivered)

Vite + React project initialized and running

React Router DOM integrated with base routes + layout

React Query configured globally (QueryClient + Devtools optional)

Feature based folder structure for pages, components, hooks, api, and routes

Environment config for API base URL

Basic UI shell (layout + navigation placeholder)

Error handling, loading states, and a “Not Found” page

Lint/format scripts (optional but recommended)

Deliverables

Repository with working dev server

/ Home page rendered through Router

At least these routes:

/ (Home)

/login (placeholder)

/dashboard (placeholder, protected-ready)

(404)
React Query provider set up in main.tsx

Example query hook (e.g., useHealthCheckQuery) hitting /health (or mock)

.env support (e.g., VITE_API_BASE_URL)

README “How to run” section updated

User stories

As a developer, I can run the project locally with one command.

As a developer, I can add new pages and routes quickly using a consistent pattern.

As a developer, I can call backend APIs using React Query with caching and error states.
